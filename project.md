# Ceefax Firefox Extension - Project Plan

A Firefox extension (New Tab override) that recreates the Ceefax teletext experience: 40×25 character grid, 8-color palette, page-number navigation, and live-ish content (news, weather, ticker, clock).

**Status:** Planning complete. No code written yet. Start at Phase 1.

---

## Architecture decisions (locked in)

| Decision | Choice |
|---|---|
| Display surface | New Tab override (`chrome_url_overrides.newtab`) |
| Manifest | MV3 |
| Rendering | CSS Grid of `<span>` cells (not canvas) - faster to build/debug |
| Grid size | 40 columns × 25 rows |
| Palette | 8-color: black, red, green, yellow, blue, magenta, cyan, white |
| Font | Bitmap teletext font (e.g. Teletext2) |
| Reader Mode | Not using native API - bundling Mozilla's Readability.js directly, run in background script against fetched HTML |
| Page numbers | 100 = index, 101–1xx = news (subpages), 200 = weather (hub) / 210–267 = per-city forecast+day-detail, 300 = ticker, 400 = Reddit (hub) / 401–409 = subreddits, 800 = clock. See decisions log 2026-07-13 for the full breakdown. |
| Data fetching | `background.js` service worker, `alarms` API on interval, cached in `browser.storage.local` |

- **2026-07-14:** Fixed a dead-link bug on page 101: `buildHeadlinesPage` listed every RSS item as a numbered selectable link (up to 20), but `background.js` only ever built full article pages for the first `ARTICLE_COUNT` (5) of them - so headline #6 onward always resolved to "PAGE NOT AVAILABLE" once the Guardian feed (routinely 20+ items) had been fetched. Fixed by passing the already-sliced 5-item list into the page builder instead of the full feed.

No code change from this alone; `refreshReddit`'s per-subreddit try/catch already means a 429 on one sub doesn't block the others, and a missing reddit page correctly falls back to "not available" (not a crash) once `lastUpdated` is set. - **2026-07-14:** Rebuilt News as a hub + multiple topic sections, mirroring the existing Weather hub/city pattern, instead of a single Guardian `/uk/rss` feed. `data.js` gains a `NEWS_TOPICS` array (World news 110, UK news 120, Business 130, Technology 140, Sport 150, Science 160 - each an actual Guardian section RSS feed, verified live with `curl` before wiring in, including following the `business`/`technology`/`sport` redirects to their `/uk/*` canonical URLs) and `ARTICLES_PER_TOPIC = 5`. Page 101 is now `buildNewsHubPage()` (topic list, same shape as `buildWeatherHubPage`); each topic's headline list lives at `topic.base` (`buildTopicHeadlinesPage`, replacing the old single `buildHeadlinesPage`), with that topic's 5 articles at `topic.base+1..+5`. `background.js`'s `refreshNews` loops `NEWS_TOPICS`, fetching each section's feed independently (one topic's failure doesn't block the others, same pattern as weather/reddit); the ticker (page 300) now joins headlines across all topics instead of just one feed. `nav.js`'s `KNOWN_DYNAMIC_PAGES` updated to a `NEWS_TOPIC_BASES`/`ARTICLES_PER_TOPIC` base+offset scheme (mirroring how weather/reddit are already computed) instead of the old flat `ARTICLE_PAGE_START`/`ARTICLE_COUNT` pair. Page 100's index label changed from "News Headlines" to "News" to match the hub semantics. Not yet manually verified in a real Firefox profile (same standing gap as everything else) - `web-ext lint` still 0 errors.
- **2026-07-14:** User-reported bug: reddit subreddit pages were rendering with zero posts even when a fetch had "succeeded". Root cause: `fetch()` doesn't reject on a 429 (or any non-2xx) - `refreshReddit` was passing the empty/error body straight to `parseRedditFeed()`, which silently returned zero entries, and that empty result then overwrote whatever good content was cached from the last successful cycle. Fixed by checking `res.ok` and throwing before parsing, so a 429 now leaves the previous cycle's cached page untouched instead of blanking it. Separately, live testing (`curl` with `x-ratelimit-*` headers) revealed reddit's rate limit is **per-IP, not per-subreddit**: any single request drops `x-ratelimit-remaining` to 0 for ~44-60s regardless of which endpoint it hit. At the old `REDDIT_FETCH_DELAY_MS = 800`, only the first of 9 sequential requests could ever succeed in a cycle - confirmed by re-testing all 9 configured subreddits back-to-back and getting 429 on every one. Raised the delay to 65000ms (verified live: two requests 65s apart both returned clean 200s) - worst case adds ~8.7 min to the refresh cycle, still under the 15-minute alarm period.
- **2026-07-14:** User-reported: no way to "open" a reddit post from its subreddit listing page - `buildSubredditPage` never populated `selectableItems`, so arrow-key selection (added 2026-07-13 for every other list page) silently did nothing there. Rather than reversing the earlier decision to skip per-post Readability extraction (out of scope - see 2026-07-13 entry above), post rows are now selectable with `{ row, url }` (the real reddit.com permalink from the Atom feed's `<link href>`, confirmed absolute via a live fetch) instead of `{ row, page }`. `nav.js`'s `activateSelection` now branches: an item with `.url` calls `browser.tabs.create({ url })` to open the real thread in a new tab; an item with `.page` still calls `navigateTo()` as before. No manifest permission needed - `tabs.create()` doesn't require the `tabs` permission when the extension isn't reading back other tabs' properties.
- **2026-07-14:** Added nested sub-categories to News per user request, going from a flat 6-topic list straight to articles, to a 3-tier hub: topic hub (page 100-based, e.g. 110) -> 3 subcategories (e.g. World news' Europe/Americas/Asia, `base+1..+3`) -> that subcategory's 3 articles (`base+4..+12`, 9 total per topic). Topic bases moved from the tightly-packed 110/120/.../160 to 110/125/140/155/170/185 (spacing of 15) so each topic's 13-slot range (1 hub + 3 subcategory lists + 9 articles) can't reach into the next topic's or into 200 (weather). All 18 subcategory RSS URLs (3 per topic x 6 topics - Europe/Americas/Asia, Politics/Education/Society, Economics/Banking/Markets, Internet/Games/Mobile phones, Football/Cricket/Rugby union, Space/Physics/Genetics) were verified live via `curl` before wiring in, including following `games`' and `rugby-union`'s canonical-URL redirects. Article count per subcategory dropped from the old flat scheme's 5 to 3, to keep each topic's page range small enough to fit the 15-number spacing and keep the total Guardian fetch count per cycle reasonable (18 RSS feeds + up to 54 article fetches, vs. the previous 6 + 30). `background.js`'s `refreshNews` now has a nested loop (topic -> subcategory); `buildTopicHeadlinesPage` replaced by `buildTopicHubPage` (lists subcategories) + `buildSubcategoryHeadlinesPage` (lists that subcategory's articles, structurally identical to the old headline-list page). Verified end-to-end with a live jsdom dry run against the real Guardian feeds: all 80 news pages built, every hub -> subcategory -> article link resolved, 0 broken links.
- **2026-07-14:** User-reported regression, caused by the same day's reddit-delay fix above: every page started showing "FETCHING LATEST DATA" persistently. Root cause: `refreshAll()` only ever wrote to `storage.local` **once, at the very end**, after `refreshNews`, `refreshWeather`, *and* `refreshReddit` had all finished sequentially. Widening `REDDIT_FETCH_DELAY_MS` to 65000 stretched that final reddit phase to ~8.7 min worst case, so news and weather - which finish in seconds - were now silently held hostage behind reddit's pacing on every single refresh, not just the first one. Fixed by extracting a `persistPages()` helper and writing after each of the three phases (news, then weather, then reddit) instead of batching everything into one write; `lastUpdated` is now only set at the very end so the loading-vs-not-found distinction for not-yet-fetched pages is preserved. Went a step further for reddit specifically, since its own phase is the one that's now genuinely slow: `refreshReddit` was refactored to persist each subreddit immediately after its own fetch (and the hub last), so subreddit pages appear progressively as they're ready (~65s apart) rather than all 9 arriving together only after the full ~8.7 min phase completes. Verified with a fast mocked-fetch jsdom test (real timers/network would take the full ~9 min): confirmed news writes first, weather second, then a run of incremental per-subreddit writes, with `lastUpdated` written last of all 13 calls - matching the intended write order.
